webpack常用loader与plugin

2020-06-14 | Webpack

Webpack 常用属性

mode

用来配置是开发模式还是生产模式，可以对代码进行压缩等操作

entry

设置文件入口

output

设置文件出口
filename 为打包后的名字，path 可以配置打包输出路径

webpack 打包原理，从 entry 开始，以路径作为 key，value 为打包后的 js 立即执行函数，放入缓存对象中，遇到引用后继续递归打包，有缓存则跳过，无缓存就打包。

–config 后可以跟自己自定义的 webpack 名字。

externals

申明该下面的模块都是外部引入的，不需要额外的打包。

optimization

优化项

devServer

配置本地静态服务，也可以配置代理来解决跨域的问题。

plugins

配置插件，数组。

resolve

主要用来配置解析第三方包的规则。
modules 配置路径解析第三方包时，只在指定目录查找。
alias: 引入的时候可以取别名
mainFiles:定义主入口文件
extensions:文件可以省略扩展名，没找到定义的后缀就在数组中往后找

module

module 下的 rules 是个数组，可以放对应的文件与处理的 loader，单个 loader 是字符串，多个 loader 可以用数组，执行顺序从右往左，从下往上

css-loader

负责解析@import

style-loader

将 css 插入到 head 标签中

less-loader

将 less 解析成 css

autoprefixer

自动给 css 加前缀，在 loader 中引入 postcss，然后在 postcss 文件中引入 autoprefixer

babel-loader

把高版本的 js 转为低版本的，如 es6 转为 es5
代码运行时，高级语法正常运行需要融入 plugin-transform-runtime

eslint-loader

校验语法规范，错误提示。需要加 enforce “pre”来保证在其他 loader 之前进行

expose-loader

全局 loader 比如 jquery 暴露给全局 window.$

file-loader

默认在内部生成一张图片到配置的 output 目录下，把生成的图片的名字返回回来。

url-loader

将图片专程 base64 返回回来，可以减少 http 请求，但是图片的大小会比原来大。可以更改图片输出路径。

html-withimg-loader

支持 html 中通过<img>来引入图片

plugin

html-webpack-plugin

自动生成 html 模版,
参数： template,filename,minify(压缩)

mini-css-extract-plugin

将 css 打包到 link 标签中(代替原来在 style 中)

optimize-css-assets-webpack-plugin

压缩 css，可以更改路径，在优化项的 minimizer 中

uglifyjs-webpack-plugin

压缩 js，在优化项的 minimizer 中

webpack.ProvidePlugin

在每个模块中国呢注入某个外部模块，就不需要再引入了，如 jquery 的参数写成$:jquery

clean-webpack-plugin

每次打包的时候清除之前的文件。

copy-webpack-plugin

可以把其他文件夹的内容拷贝到打包目录下

webpack.BannerPlugin

版权声明插件

webpack.DefinePlugin

定义环境变量，如生产环境，测试环境，字符串用 JSON.stringify,其他类型直接单引号包裹就行

webpack-merge

公共配置放在 base 中，dev 引入 base 并配置开发独有的的，prod 引入 base 并开发 prod 独有的

webpack 优化

noParse

不去解析某个库的依赖库

webpack.ignorePlugin

忽略指定位置的某个引入

webpack.dllPlugin

把不怎么变化的库打包成 dll 引入，减少打包本体的体积。
DllReferencePlugin 先去 dll 中找，找不到再打包

happypack

运用多线程打包，加快打包速度。

webpack 自带优化

import 引入的文件，当没有使用的情况下，在生产模式下不会打包进来，这个叫 tree-shaking。require 的文件就会打包进来。

抽离公共模块

当多入口应用引用同一个模块的时候，可以抽取公共模块缓存。单页应用不需要。optimization 下的 splitChunks 用来分割模块

懒加载

import() 语法，原理是使用 jsonp 实现动态加载文件。

热更新

webpack-dev-server hot 配置为 true
webpack.hotModuleReplacementPlugin 热更新
webpack.NamedModulesPlugin 打印更新模块路径
热更新不用刷新页面
